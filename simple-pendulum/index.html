Simple pendulum¶

Author: Ludovic Charleux (ludovic.charleux@univ-smb.fr)

Background¶

This tutorial aims to model and simulate the oscillations of a simple pendulum. A representation is given below (source: Wikipedia).

The pendulum consists of a point mass mmm located at the point MMM attached to a rigid arm considered without mass. The length of the arm is noted LLL. This arm is articulated with respect to a frame considered as an inertial reference frame. The angle between the arm and the vertical direction is noted θ\thetaθ. A simple modeling using dynamics leads to :

mA⃗(M/0)=P⃗+T⃗m\vec A(M/0) = \vec P + \vec TmA(M/0)=P+T
(1)

Where:

  • A⃗(M/0)\vec A(M/0)A(M/0) is the acceleration of the mass,

  • P⃗\vec PP if the weight of the mass,

  • T⃗\vec TT if the reaction force of the arm.

A projection of this equation along the direction perpendicular to the arm gives a more simple equation:

θ¨=−gLsin⁡θ\ddot \theta = -\dfrac{g}{L} \sin \thetaθ¨=−Lg​sinθ
(2)

This equation is a non linear ODE of the second order whose exact solution is not known. It is however possible to assume that θ\thetaθ is small enough to write that sin⁡θ≈θ\sin \theta \approx \thetasinθ≈θ. In this case, the equation comes down to the one of a free and undamped harmonic oscillator treated in the theoretical part. The object of this tutorial is to solve this problem by a numerical approach in a general way without simplifying assumptions.

Part 2: solver validation at small angles¶

In this first part, we want to validate the operation of the numerical solvers by comparing their results to a known exact solution. To do so, we will focus on the case of the simple pendulum oscillating at small angles. In this case, the sine can be approximated by sin⁡θ≈θ\sin \theta \approx \thetasinθ≈θ and the equation of the pendulum can be reduced to that of a harmonic oscillator. The exact solution is then:

x(t)=θ0cos⁡(ω0t)x(t) = \theta_0 \cos(\omega_0 t)x(t)=θ0​cos(ω0​t)
(5)

With:

  • X0=[θ0,0]TX_0 = [\theta_0, 0]^TX0​=[θ0​,0]T

def Euler(func, X0, t):
    """
    Euler integrator.
    """
    dt = t[1] - t[0]
    nt = len(t)
    X = np.zeros([nt, len(X0)])
    X[0] = X0
    for i in range(nt - 1):
        X[i + 1] = X[i] + func(X[i], t[i]) * dt
    return X


def RK4(func, X0, t):
    """
    Runge and Kutta 4 integrator.
    """
    dt = t[1] - t[0]
    nt = len(t)
    X = np.zeros([nt, len(X0)])
    X[0] = X0
    for i in range(nt - 1):
        k1 = func(X[i], t[i])
        k2 = func(X[i] + dt / 2.0 * k1, t[i] + dt / 2.0)
        k3 = func(X[i] + dt / 2.0 * k2, t[i] + dt / 2.0)
        k4 = func(X[i] + dt * k3, t[i] + dt)
        X[i + 1] = X[i] + dt / 6.0 * (k1 + 2.0 * k2 + 2.0 * k3 + k4)
    return X
